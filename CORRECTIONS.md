# Corrections Log

Every substantive correction to the Frontier Women Index is logged here with date, nature, and resolution. This log is append-only: corrections are not edited or removed after being recorded.

The purpose of this file is accountability. Research infrastructure is only trustworthy if it openly acknowledges when it was wrong.

## Format

Each entry follows this structure:

```
## YYYY-MM-DD · <Leader Name> or <Global Issue>

**Nature:** Brief description of what was wrong.
**Reported by:** GitHub handle of reporter, or "internal review" if caught by maintainers.
**Resolution:** What changed, with a link to the PR.
**Source diff:** `data/leaders/<slug>.yaml` (link to commit).
```

## When a correction triggers a version bump

Minor corrections (typos, link updates, citation-count snapshots) do not bump the version. Substantive corrections (tier changes, role changes, removal of a claim) bump the patch version (1.0 → 1.0.1). Methodology changes bump the minor version (1.0 → 1.1). See [`METHODOLOGY.md`](./METHODOLOGY.md#versioning) for the full versioning scheme.

## When a correction is held back from this log

Corrections are logged publicly with one narrow exception: when a correction resolves a safety-sensitive situation (for example, a leader's address or personal contact information was accidentally included and needs to be removed, or a leader is facing harassment and removal is requested for personal safety). In those cases, the log will note that a correction occurred, the date, and that details are withheld for safety reasons, without revealing the specifics.

No other reason is sufficient to withhold a correction from the public log.
